# 貢獻指南
在你推薦工具時，為了方便整體的資料管理，請協助依照下面的步驟跟要求進行。  
如果有任何問題，也可以發 Issue 詢問。

## 基本需求
推薦的所有工具必須符合以下條件：
1. 必須是對大眾公開，任何人都可以使用的
2. 必須支援 **Artale (繁體中文版)**

## 貢獻步驟
1. Fork 這個專案
2. 修改 `README.md`，新增你推薦的工具  
修改時，請務必符合下面的[**標準格式**](#%E5%B7%A5%E5%85%B7%E6%A8%99%E6%BA%96%E6%A0%BC%E5%BC%8F)
3. 發起新的 [Pull Request](https://github.com/vongola12324/awesome-artale/compare)  
發PR時，請務必填寫樣板要求的內容，可以使用中文或英文進行描述

## 工具標準格式
每個工具都應該按照以下格式來呈現。
```
- [工具名稱](工具網址) - 工具描述
```

其中，**工具網址**必須是任何人皆可以連到的，不可以是內部網址或是測試網址。  
如果，有同一個工具網站提供兩個以上的工具時，請將工具網站放在第一層、實際的工具連結放置第二層，範例如下：
```
- [工具集網站](網站網址)
    - [A工具名稱](A工具網址) - A工具介紹
    - [B工具名稱](B工具網址) - B工具介紹
```
一個**工具**可以放置於不同的**分類**下，如果你發現現有**分類**無法正確分類這個工具，你可以自行新增，格式如下：
```
## 分類A
- [工具名稱](工具網址) - 工具描述
```

## PR檢查步驟
所有的 PR 都會由維護者依據下面的步驟進行檢查：
1. 格式是否符合前面的標準格式
2. 工具是否符合前面的基本需求

如果你的格式不正確，清單的維護者可能會要求你進行修改。

## 拒絕合併
有下面情況的話，你的推薦可能會被否決。
1. 如果先前已經有人推薦過了，那麼後續重複的推薦PR會被標註並關閉
2. 推薦的工具不符合基本需求
3. PR內容格式不正確且無法配合調整
4. 其他導致這個列表無法正常呈現的調整
